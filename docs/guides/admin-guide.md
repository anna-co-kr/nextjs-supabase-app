## 1. 로그인 및 인증

### 접속 방법

`/admin-login` 경로로 이동하여 환경변수에 설정된 비밀번호를 입력합니다.

```
https://your-domain.com/admin-login
```

### 세션 관리

- 로그인 세션은 서버 측 쿠키로 관리됩니다.
- 브라우저를 닫아도 세션이 유지됩니다.
- 로그아웃 시 우측 상단의 **로그아웃** 버튼을 클릭하세요.

### 보안 주의사항

- 공용 컴퓨터 사용 후 반드시 로그아웃하세요.
- 비밀번호는 최소 8자 이상의 강력한 값을 사용하세요.
- 미인증 접근 시 자동으로 로그인 페이지로 리다이렉트됩니다.

## 2. 대시보드

`/admin` 경로의 메인 대시보드에서 다음 항목에 빠르게 접근할 수 있습니다.

| 카드            | 설명                            |
| --------------- | ------------------------------- |
| 견적서 관리     | 발행된 모든 견적서 조회 및 관리 |
| 클라이언트 관리 | 클라이언트 정보 확인            |
| 최근 활동       | 최근 견적서 발행 및 승인 현황   |

좌측 사이드바에서 **견적서 목록**을 클릭하여 전체 목록으로 이동할 수 있습니다.

## 3. 견적서 목록 관리

`/admin/invoices`에서 노션 데이터베이스의 모든 견적서를 조회할 수 있습니다.

### 테이블 컬럼 설명

| 컬럼         | 설명                                     |
| ------------ | ---------------------------------------- |
| 견적서 번호  | 고유 견적서 식별 번호 (예: INV-2026-001) |
| 클라이언트명 | 견적서 수신 클라이언트 이름 및 회사명    |
| 발행일       | 견적서 발행 날짜                         |
| 유효기간     | 견적서 유효 만료일                       |
| 총액         | 견적서 합계 금액 (원화)                  |
| 상태         | 현재 견적서 상태 (아래 참조)             |
| 링크         | 클라이언트 공유용 URL                    |
| 작업         | 견적서 보기 버튼                         |

### 견적서 상태 종류

| 상태     | 의미                                   |
| -------- | -------------------------------------- |
| 대기     | 클라이언트에게 아직 공유되지 않은 상태 |
| 공유완료 | 클라이언트에게 링크가 공유된 상태      |
| 확정     | 클라이언트가 견적을 확정한 상태        |
| 취소     | 견적이 취소된 상태                     |

### 정렬

- **발행일** 컬럼 헤더 클릭 → 발행일 기준 정렬
- **총액** 컬럼 헤더 클릭 → 금액 기준 정렬

### 페이지네이션

- 기본 10건씩 표시
- 하단 **이전 / 다음** 버튼으로 페이지 이동

## 4. 검색 및 필터링

### 텍스트 검색

상단 검색창에 다음 항목으로 검색할 수 있습니다.

- **클라이언트명** (예: `김민준`)
- **견적서 번호** (예: `INV-2026-001`)

검색어 입력 후 자동으로 결과가 필터링됩니다. 검색 초기화는 검색창 우측 ✕ 버튼을 클릭하세요.

### 고급 필터

우측 상단 **필터** 버튼을 클릭하면 다음 필터를 적용할 수 있습니다.

- **상태 필터**: 대기 / 공유완료 / 확정 / 취소
- **날짜 범위**: 발행일 기준 시작일 ~ 종료일 지정

## 5. 링크 관리

각 견적서마다 클라이언트에게 전달할 수 있는 고유 URL이 자동 생성됩니다.

### 링크 복사

1. 견적서 목록에서 링크 컬럼의 **복사 버튼(📋)** 클릭
2. "링크가 복사되었습니다" 토스트 알림 확인
3. 클라이언트에게 붙여넣기로 전달

### 링크 공유

**공유 버튼(↗)** 클릭 시 다음 방법으로 공유할 수 있습니다.

| 방법                   | 설명                                                          |
| ---------------------- | ------------------------------------------------------------- |
| 이메일 공유            | 기본 메일 앱으로 견적서 링크가 포함된 이메일 작성             |
| 텔레그램 공유          | 텔레그램으로 링크 공유                                        |
| 모바일 (Web Share API) | iOS/Android에서는 네이티브 공유 시트로 카카오톡, 문자 등 공유 |

### 링크 직접 열기

링크 컬럼의 **외부 링크 아이콘(↗)** 클릭 시 새 탭에서 견적서 조회 페이지가 열립니다.

## 6. 다크모드 설정

우측 상단의 **테마 전환 버튼(☀️/🌙)** 을 클릭하여 테마를 선택할 수 있습니다.

| 옵션   | 설명                     |
| ------ | ------------------------ |
| 라이트 | 밝은 흰색 계열 테마      |
| 다크   | 어두운 검정 계열 테마    |
| 시스템 | OS 설정에 따라 자동 적용 |

선택한 테마는 브라우저 로컬스토리지에 저장되어 다음 방문 시에도 유지됩니다.

## 7. 환경 변수 설정

`.env.local` 파일에 다음 변수를 반드시 설정하세요.

```bash
# Notion API
NOTION_API_KEY=secret_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx  # 정확히 32자

# 관리자 인증
ADMIN_PASSWORD=your-secure-password-here             # 최소 8자
SESSION_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx       # 정확히 32자

# 배포 URL (프로덕션 시 필수)
NEXT_PUBLIC_BASE_URL=https://your-domain.com
```

### SESSION_SECRET 생성 방법

```bash
# 터미널에서 32자 랜덤 문자열 생성
openssl rand -hex 16
# 또는
node -e "console.log(require('crypto').randomBytes(16).toString('hex'))"
```

## 8. 프로덕션 배포 체크리스트

Vercel 또는 기타 플랫폼 배포 전 아래 항목을 확인하세요.

### 환경 변수

- [ ] `NOTION_API_KEY` 설정 완료
- [ ] `NOTION_DATABASE_ID` 설정 완료 (32자 확인)
- [ ] `ADMIN_PASSWORD` 강력한 비밀번호로 설정
- [ ] `SESSION_SECRET` 정확히 32자 랜덤 문자열 설정
- [ ] `NEXT_PUBLIC_BASE_URL` 실제 도메인으로 설정

### 보안

- [ ] 약한 비밀번호(`admin1234`, `password` 등) 미사용 확인
- [ ] 기본 `SESSION_SECRET` 미사용 확인
- [ ] HTTPS 적용 확인 (HSTS 헤더 동작 전제)
- [ ] 관리자 URL(`/admin`)이 공개 색인되지 않도록 `robots.txt` 확인

### 빌드 및 동작

- [ ] `npm run build` 오류 없이 완료
- [ ] `npm run check-all` 모든 검사 통과
- [ ] 관리자 로그인 정상 동작 확인
- [ ] 견적서 목록 조회 정상 동작 확인
- [ ] 클라이언트 견적서 조회 페이지 정상 동작 확인
- [ ] 링크 복사 기능 동작 확인

### 성능

- [ ] 견적서 조회 페이지 로드 < 3초
- [ ] 관리자 대시보드 로드 < 2초
- [ ] Notion API 캐싱(60초) 정상 동작 확인

### Vercel 배포

```bash
# Vercel CLI 배포
vercel --prod

git push origin main
```
