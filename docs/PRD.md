# Gather MVP PRD

## 핵심 정보

**목적**: 소규모~중규모 모임 주최자가 공지·참여자 관리·정산을 메신저·스프레드시트·계좌이체로 파편화해서 처리하는 반복 작업을 하나의 웹 서비스로 통합하여 부담을 제거한다
**사용자**: 정기 모임(수영·헬스 등 5~20명) 또는 일회성 이벤트(여행·파티)를 운영하는 주최자와 해당 이벤트의 참여자

---

## 사용자 여정

```
1. 랜딩 페이지
   ↓ 회원가입 / 로그인 버튼 클릭

2. 회원가입 페이지 / 로그인 페이지
   ↓ 인증 완료

3. 대시보드 페이지 (내 이벤트 목록)
   ↓
   [주최자] 이벤트 생성 버튼 클릭 → 이벤트 생성 페이지
   [참여자] 공유 링크 수신 → 이벤트 상세 페이지 (공개)

4-A. 이벤트 생성 페이지 (주최자)
   ↓ 저장 완료
   → 이벤트 관리 페이지로 이동

4-B. 이벤트 상세 페이지 (공개, 참여자)
   ↓ 참여 신청 버튼 클릭
   → 로그인 상태: 참여 신청 페이지
   → 비로그인 상태: 로그인 페이지 → 참여 신청 페이지

5-A. 이벤트 관리 페이지 (주최자)
   ↓ 하위 탭 선택
   → 참여자 관리 페이지
   → 공지 관리 페이지
   → 정산 관리 페이지

5-B. 참여 신청 페이지 (참여자)
   ↓ 신청 완료
   → 이벤트 상세 페이지 (신청 완료 상태)
   → 이후 주최자 승인 → 공지 목록 페이지 / 정산 현황 페이지 접근 가능

6. 완료
   → 주최자: 참여자 관리·공지 작성·정산 등록 반복
   → 참여자: 공지 확인·정산 금액 확인 반복
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                     | 설명                                                                  | MVP 필수 이유                   | 관련 페이지                            |
| -------- | -------------------------- | --------------------------------------------------------------------- | ------------------------------- | -------------------------------------- |
| **F001** | 이벤트 CRUD                | 이벤트명·설명·장소·날짜/시간·유형·최대 인원·마감일 입력 및 수정·삭제  | 서비스 핵심 데이터 단위         | 이벤트 생성 페이지, 이벤트 관리 페이지 |
| **F002** | 이벤트 공개 링크 생성      | 비로그인 사용자도 이벤트 정보를 볼 수 있는 공개 URL 발급              | 참여자 초대의 유일한 진입점     | 이벤트 생성 페이지, 이벤트 상세 페이지 |
| **F003** | 참여 신청                  | 참여자가 이벤트에 참여를 신청하고 주최자가 확정·대기·거절 상태를 관리 | 모임 구성의 핵심 플로우         | 참여 신청 페이지, 참여자 관리 페이지   |
| **F004** | 대기자 명단 자동 배치      | 정원 초과 신청 시 자동으로 대기 상태로 전환                           | 정원 초과 상황 처리의 필수 UX   | 참여 신청 페이지, 참여자 관리 페이지   |
| **F005** | 참여자 일괄 내보내기       | 확정·대기·거절 참여자 목록을 CSV로 다운로드                           | 주최자 외부 관리 도구 연계 지원 | 참여자 관리 페이지                     |
| **F006** | 공지 작성 및 고정          | 제목+본문 공지 작성, 고정 공지 설정, 댓글/이모지 리액션               | 메신저 공지 대체 핵심 기능      | 공지 관리 페이지, 공지 목록 페이지     |
| **F007** | D-1·D-3 자동 리마인더      | 이벤트 1일·3일 전 참여 확정 멤버에게 알림 자동 발송                   | 주최자 수동 공지 횟수 감소      | 공지 관리 페이지                       |
| **F008** | 정산 등록 및 분담 계산     | 항목명+금액 다중 입력, 균등/개별/비율 분담 방식 선택                  | 비용 정산 파편화 해결           | 정산 관리 페이지                       |
| **F009** | 납부 확인 및 미납 리마인더 | 주최자가 납부 완료를 체크하고 미납자에게 리마인더 발송, 계좌번호 등록 | 정산 마무리 루프 완성           | 정산 관리 페이지, 정산 현황 페이지     |

### 2. MVP 필수 지원 기능

```

    ├── 로그인 - F010
    └── 회원가입 - F010

주최자 메뉴 (로그인 후 / 이벤트 생성자)
├── 대시보드 - F001, F011
│   └── 내 이벤트 목록 및 새 이벤트 생성 진입
├── 이벤트 생성 - F001, F002
└── [이벤트명] 관리
    ├── 이벤트 관리 - F001, F002
    ├── 참여자 관리 - F003, F004, F005
    ├── 공지 관리 - F006, F007
    └── 정산 관리 - F008, F009

참여자 메뉴 (로그인 후 / 이벤트 참여자)
├── 대시보드 - F011
│   └── 참여 중인 이벤트 목록
└── [이벤트명]
    ├── 이벤트 상세 (공개) - F002, F003
    ├── 공지 목록 - F006
    └── 정산 현황 - F009

공통 (모든 로그인 사용자)
└── 로그아웃
```

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) - React 풀스택 프레임워크
- **TypeScript 5.6+** - 타입 안전성 보장
- **React 19** - UI 라이브러리 (최신 동시성 기능)

### 스타일링 & UI

- **TailwindCSS v4** (설정 파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 폼 & 검증

- **React Hook Form 7.x** - 폼 상태 관리
- **Zod** - 스키마 검증 라이브러리

### 상태 관리

- **Zustand** - 경량 클라이언트 전역 상태 관리

### 백엔드 & 데이터베이스

- **Supabase** - BaaS (인증, PostgreSQL DB, RLS, Realtime)
- **PostgreSQL** - 관계형 데이터베이스 (Supabase 내장)

### 이메일 알림

- **Resend** - 트랜잭션 이메일 발송 (D-1·D-3 리마인더, 미납 리마인더)

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼
